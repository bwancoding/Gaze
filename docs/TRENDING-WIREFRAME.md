## 3. 交互流程说明

### 3.1 页面加载流程

```
用户访问 /trending
        ↓
加载页面骨架屏 (Skeleton)
        ↓
API 请求：GET /api/trending?limit=20
        ↓
接收数据 → 渲染 EventList
        ↓
显示"更新时间"
        ↓
页面就绪
```

### 3.2 筛选交互流程

```
用户选择筛选条件
        ↓
更新筛选状态 (本地 state)
        ↓
防抖 300ms
        ↓
API 请求：GET /api/trending?ideology=left&category=政治，科技&sort=heat
        ↓
刷新 EventList
        ↓
更新 URL 参数 (?ideology=left&category=政治，科技)
```

### 3.3 排序交互流程

```
用户点击排序下拉
        ↓
选择"热度"或"时间"
        ↓
立即切换排序 (前端排序或 API 请求)
        ↓
重新渲染 EventList
        ↓
更新 URL 参数 (?sort=time)
```

### 3.4 点击事件卡片流程

```
用户点击 EventCard
        ↓
记录点击事件 (埋点)
        ↓
路由跳转：/event/{eventId}
        ↓
详情页加载
```

### 3.5 加载更多流程

```
用户点击"加载更多"
        ↓
显示加载状态 (Loading spinner)
        ↓
API 请求：GET /api/trending?offset=20&limit=20
        ↓
追加新数据到 EventList
        ↓
隐藏加载状态
        ↓
若少于 20 条 → 隐藏"加载更多"按钮
```

## 4. 响应式断点设计

### 4.1 断点定义

| 断点名称 | 宽度范围 | 设备类型 |
|---------|---------|---------|
| Mobile | < 768px | 手机竖屏 |
| Tablet | 768px - 1023px | 平板竖屏/手机横屏 |
| Desktop | 1024px - 1439px | 笔记本/桌面 |
| Large Desktop | ≥ 1440px | 大屏显示器 |

### 4.2 各断点布局策略

#### Mobile (< 768px)
- 单栏布局
- 筛选面板可折叠
- 列表项紧凑（padding: 12px）
- 字体大小：标题 16px，元数据 12px
- 导航栏汉堡菜单

#### Tablet (768px - 1023px)
- 单栏布局（筛选面板固定在顶部）
- 筛选条件横向排列
- 列表项中等（padding: 16px）
- 字体大小：标题 18px，元数据 13px
- 导航栏显示部分功能

#### Desktop (1024px - 1439px)
- 双栏布局（左侧筛选，右侧列表）
- 筛选面板固定
- 列表项宽松（padding: 20px）
- 字体大小：标题 20px，元数据 14px
- 导航栏完整功能

#### Large Desktop (≥ 1440px)
- 双栏布局（内容居中，最大宽度 1600px）
- 增加两侧留白
- 列表项可显示更多信息（如摘要）
- 字体大小：标题 22px，元数据 15px

## 6. 可访问性 (A11y) 考虑

- ✅ 筛选选项使用 `<fieldset>` + `<legend>` 语义化标签
- ✅ 列表使用 `<ol>` 有序列表（排名有意义）
- ✅ 颜色对比度符合 WCAG AA 标准
- ✅ 支持屏幕阅读器（aria-label 标注热度分数等）
- ✅ 焦点状态清晰可见

## 7. 性能优化建议

1. **列表虚拟化**: Top20 可全量渲染，但"加载更多"后启用虚拟滚动
2. **图片懒加载**: 媒体 Logo 使用懒加载
3. **API 缓存**: 筛选条件变更时使用 SWR/React Query 缓存
4. **骨架屏**: 首屏加载显示骨架屏而非 Loading spinner
5. **预加载**: 鼠标 hover 事件卡片时预加载详情页数据
